git工具： 1. 版本管理: 就是可以保留文件之前的样子，可以非常方便的回到过去，或再回到现在 2. 多人协作：可以提供是否存在冲突的提示 github网站： https://github.com/ git工具的安装： https://git-scm.com/ -> 下载 git工具的使用两种形式： 1. 可视化，GIT CUI 2. 代码，GIT Bash 如何使用git工具？ 1. 指定哪些文件需要git工具去管理，找到管理文件的根目录 在根目录下输出git init 生成 .git文件夹，这样就可以维护这个目录下的所有文件了 2. 设置贡献者，让git工具知道是谁在操作git git config --list -> 查看git工具的配置信息 git config --global user.name "xxx" git config --global user.email "xxx" git config --global --unset user.name 3. 到底git去管理的文件的什么？ 管理的快照，就是一种状态status：添加、修改、删除、重命名都是状态 4. git工具是分布式的 版本管理在每台电脑中都可以操作 5. git工具的分区 1. 工作区 2. 暂存区 3. 版本区 6. git status : 查看三个区的状态的 打印的信息中包含红色字样，说明工作区跟暂存区状态不一样 打印的信息中包含绿色字样，说明暂存区跟版本区状态不一样 nothing to commit, working tree clean -> 三个区状态一样 7. git add : 把工作区的状态同步到暂存区 git add a.txt : 找到a.txt进行同步状态 git add . : 把工作区改变的状态全部同步到暂存区 8. git commit : 把暂存区的状态同步到版本区，并且生成一个版本 只能把暂存区中所有的不同状态同步到版本区 git commit -m "添加了a.txt" git commit --amend -m "修改了a" -> 当前这个新的版本会跟最近的一个版本进行合并 9. git log: 查看版本区中的所有版本的 git内容太多，如果想退出，输入 q git log --pretty=oneline -> 已一行的形式进行展示，查看更加的方便 10. 删除操作 1. 在工作区删除文件，然后add，再commit 2. git rm c.txt，自动进行了工作区和暂存区的行为，只要commit就可以了 11. 对文件进行重命名： 1. 在工作区重命名文件，然后add，再commit 2. git mv c.txt d.txt，自动进行了工作区和暂存区的行为，只要commit就可以了 12. git diff 对比的命令 git diff -> 工作区 与 暂存区 对比 git diff master -> 工作区 与 版本区中的master分支进行对比 git diff --cached -> 暂存区 与 版本区当前的分支进行对比 13. git reset --hard HEAD -> 回退到当前指针对应的版本 git reset --hard HEAD^ -> 回退到当前指针的前一个对应的版本 git reset --hard HEAD^^ -> 回退到当前指针的前两个对应的版本 git reset --hard HEAD~50 -> 回退到当前指针的前50个版本 git reset --hard [commit_id] -> 回退到指定id的版本 git reflog -> 查看操作的历史记录 14. git checkout 检出文件 可以得到之前版本中的文件状态，不影响版本的回退 git checkout commitid a.txt -> 检出文件到工作区和暂存区 git checkout -- a.txt -> 从暂存区检出文件到工作区 // ------------------------------------------------ git工具实现多人协作： github可以用来充当远程仓库，作为项目版本的一个中转站，大家都在这个远程仓库下进行通信 github上创建账号 (gitlab gitee) 邮箱激活账号 创建仓库 让电脑跟远程仓库关联在一起，才能上传版本到仓库 可以选择 HTTPS 或 SSH SSH用法： 1. .ssh文件夹 -> id_rsa(私钥) id_rsa.pub(公钥) 生成.ssh文件夹：ssh-keygen -t rsa -C "youremail@example.com" 2. 把公钥给github网站 如何把电脑中的版本传到github网站上 版本要传到指定的远程仓库上？ git remote -> 查看是否有对应的远程仓库 git remote -v -> 查看更详细的信息 git remote add origin git@github.com:ghostdp/gitstudy2.git git push origin master -> 上传mater分支到origin对应的远程仓库 如何在一台电脑中拉取github上的指定的分支 git clone -b master git@github.com:ghostdp/gitstudy2.git git clone 不需要权限，可以拉取github上所有开源的远程仓库 但是上传版本到github是需要权限的，需要在项目的settings中设置协作者 已经存在了项目，如何更新github上的版本到本地呢？ git pull origin master 如何解决冲突 1. git pull ，把有问题的文件做自动合并 2. 协商解决文件冲突问题 3. 重新提交版本，并push就可以了 // 分支： master -> 默认的 v1 v2 v3 .... git branch -> 查看当前有多少个分支 git branch new -> 创建一个new分支 git checkout new -> 切换到new分支上 在master分支下，运行这句话：git merge new 就是把new合并到master上 git branch -d new -> 删除new分支 git push origin new -> 把new分支上传到远程仓库上 git tag -> 查看里程碑 git tag 1.0.0
